# Mechanism Switching Plan

## Research Question

The target construct is the timing and stability of a learner's shift from broad
cross-situational tracking to a more PbV-like state. This should be treated as a
latent learning-mechanism profile, not as a fixed participant label.

The adopted task design is Plan 2: 20 words per participant, 5 learning
blocks, 3 encounters per word per block, and a click-based 5AFC test after each block.
Thus, each word has 15 learning encounters and 5 test observations.

## Behavioral Signature

The most direct PbV signature is response contingency for the same word:

- After a correct previous response, later accuracy should rise sharply.
- After an incorrect previous response, later accuracy should remain closer to
  chance because the correct referent was not carried forward.

An associative learner should show smoother improvement as co-occurrence evidence
accumulates, including some benefit even after previous incorrect responses.

## Recommended Model Ladder

Start simple and only move to richer latent-state models if the data support it.

1. Contingent-response summaries
   - Estimate `P(correct_t | previous_correct_t-1)`.
   - Estimate `P(correct_t | previous_incorrect_t-1)`.
   - Compute the difference overall, by block, and by phonology condition.

2. Windowed model comparison
   - Fit associative and PbV prediction models separately by early/middle/late
     windows or by block.
   - Track `delta_loglik` or `delta_BIC` over time.

3. Continuous mixture model
   - Use a trial-level mixture:

```text
P(response_t) =
  pi_t * P_PbV(response_t)
  + (1 - pi_t) * P_associative(response_t)

logit(pi_t) =
  block + previous_correct + phonology + contrast + participant random effect
```

   - Interpret `pi_t` as the probability that the response is generated by a
     PbV-like mechanism.

4. Markov-switching model
   - Use only after the above models show stable evidence for discrete state
     changes.
   - Candidate latent states:
     - `explore`
     - `associative`
     - `pbv`
   - Let transition probabilities depend on block, previous correctness,
     phonology, and participant-level random effects.

## Candidate Aptitude Indices

- `pbv_onset_block`: first block where posterior PbV probability exceeds a
  preregistered threshold.
- `pbv_stability`: probability of staying in PbV after entering it.
- `assoc_to_pbv_transition`: estimated transition tendency from associative to
  PbV state.
- `hard_delay`: difference in PbV onset between easy and hard phonology items.
- `contingency_delta`: accuracy after previous correct minus accuracy after
  previous incorrect.
- `switching_flexibility`: degree to which state probabilities adapt to stimulus
  difficulty rather than remaining fixed.

## Design Implications

The task needs enough repeated observations per word. A final forced-choice test
alone is insufficient. The learning phase should collect one response after each
word presentation, with no feedback, and should preserve word-specific trial
history in the export.

For Plan 2, switch timing should primarily be estimated over `encounterIndex`
1 to 15 rather than only over block 1 to 5. Blocks are useful for summaries, but
encounter-level indexing gives the model enough temporal resolution.
